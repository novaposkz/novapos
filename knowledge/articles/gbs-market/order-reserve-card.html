Карточка заказа/резерва

В кассовой программе GBS.Market доступна возможность по созданию заказов/резервов

Главная > База знаний > Карточка заказа/резерва

В кассовой программе GBS.Market доступна возможность по созданию заказов/резервов.

Заказ/резерв можно использовать для таких целей:

  • оформление заказа на товар, которого нет в наличии
  • резервирование какого-то товара за покупателем
  • фиксация предоплаты за товары или услуги

На скриншоте выше показан внешний вид карточки заказа/резерва.

Открыть карточку заказа/резерва можно из меню главной формы: Документы - Новый заказ/резерв.

Панель свойств

В верхней части формы расположена панель свойств.

Контакт

Контакт - это клиент (покупатель), который оформляет заказ. Нажмите на Контакт, чтобы выбрать клиента через форму поиска контактов.

Указание контакт обязательно.

Статус

Свойство "Статус" может принимать одно из значений:

  • Открыт - заказ еще находится в работе. Такие заказы могут быть отредактированы
  • Выполнен (закрыт) - заказ уже завершен. Закрытые заказы недоступны для редактирования

При изменении статуса заказа с "Открыт" на "Выполнен" и последующем сохранении, его содержимое будет перемещено в чек (корзину) на главной форме для окончательного оформления взаиморасчетов с клиентом.

Актуален до...

Дата, до которой актуален заказ. Свойство носит только информационный характер. Заказы, которые вышли за рамки указанной даты, автоматически не изменяются.

Зарезервировать товары

Если опция включена, то при сохранении заказа/резерва для товаров, входящих в его состав, количество будет уменьшено на указанное в заказе. Например, может быть использовано, когда покупатель просит отложить какой-то товар и обещает вернуться на следующий день.

Если опция выключена, то остатки товаров, входящих в состав заказа/резерва, не будут изменяться при сохранении документа. Товары будут списаны в момент закрытия такого заказа. Например, может быть использовано для оформления заказа от покупателя на товары, которых нет в наличии.

Сохранять цены на товары

Если опция включена, то на товары, входящие в состав заказа/резерва, будут сохранены цены, актуальные на момент добавления таких товаров в документ. Например, это актуально в случаях, когда с покупателем согласована цена на товар и он уже внес полную предоплату.

Если опция выключена, то будет использоваться актуальная цена на товары. Например, это может быть полезно в случаях, когда клиент оформляет заказ товара, но конечная стоимость такого товара заведомо неизвестна.

Комментарий

Комментарий к документу. Например, адрес доставки или другая дополнительная информация.

Список товаров

На вкладке "Список товаров" отображаются товары, добавленные в заказ/резерв. Вы можете управлять товарами в списке:

  • добавить товары
  • изменить количество товаров
  • устанавливать скидку на товар
  • удалять товары из списка

В списке отображается информация:

  • штрихкод
  • категория товара
  • наименование товара
  • количество товара
  • цена
  • сумма
  • и т.д.

Кнопка "Добавить"

Нажмите на кнопку "Добавить", чтобы добавить товар в список из окна поиска товаров. Если необходимо создать новый товары - это можно сделать в окне поиска товаров.

Кнопка "Количество"

Выберите один или несколько товаров и нажмите на кнопку "Количество", чтобы указать количество для них.

Кнопка "Скидка"

Выберите один или несколько товаров в списке и нажмите на кнопку "Скидка", чтобы указать скидку на них.

Кнопка "Удалить"

Выберите один или несколько товаров и нажмите "Удалить", чтобы удалить эти товары из документа.

Строка итогов

В нижней части списка товаров отображается строка с итогами:

  • количество наименований
  • общее количество товаров в документе
  • розничная сумма товаров

Список платежей

Список платежей позволяет управлять платежами, сделанными клиентом (покупателем) по заказу/резерву:

  • добавление платежей
  • удаление платежей

Например, добавление платежа может быть полезно в случае оформления предоплаты за товар, которого нет в наличии.

Кнопка "Добавить"

Нажмите на кнопку, чтобы добавить платеж (предоплату) в заказ/резерв. Если в программе настроены способы оплаты, то вы сможете указать, каким способом (например, картой или наличными) осуществляется платеж.

Кнопка "Удалить"

Выберите один или несколько платежей и нажмите кнопку "Удалить", чтобы удалить эти платежи.

Строка итогов

В строке итогов под списком платежей отображается информация:

  • количество платежей
  • Оплачено - общая сумма платежей, внесенных по документу
  • Задолженность - остаток суммы, которую необходимо оплатить для полного расчета по документу

Сохранение заказа/резерва

В нижней части карточки заказа/резерва расположены кнопки "Сохранить" и "Отмена".

Если статус заказа изменен с "Открыт" на "Выполнен", то содержимое заказа будет перемещено в корзину для завершения взаиморасчетов.

Вопросы и ответы

Можно ли в программе оформить предоплату за товары

Да, в кассовой программе GBS.Market есть возможность оформления предоплаты (аванса). Например, это может быть полезно в случаях, когда покупатель хочет оформить заказ на товар, которого нет в наличии.

Оформить предоплату на товары можно следующим образом:

  1. откройте карточку заказа/резерва
  2. выберите клиента (покупателя), который вносит предоплату
  3. на вкладке "список товаров" добавьте товары, за которые вносится предоплата
  4. на вкладке "список платежей" добавьте платеж на сумму предоплаты
  5. сохраните заказ/резерв

Будет ли напечатан чек на предоплату?

В случае, если к программе подключена касса, программа распечатает чек согласно 54-ФЗ и ФФД 1.2 на предоплату (аванс). При этом в чеке не будет перечня товаров, а будет лишь указан номер заказа, т.к. на момент предоплаты список товаров может быть неполным.
